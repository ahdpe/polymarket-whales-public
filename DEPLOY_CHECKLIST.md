### 3. Проверка после перезапуска (первые 5 минут)

Откройте логи и проверьте наличие этих строк:

**При старте:**
```
INFO - Loaded mute state: 0 muted, 0 streaks
```
*(или число > 0, если уже были замученные)*

**Если chat_id 1580869819 получает ошибку:**
```
WARNING - 🔇 MUTED chat_id=1580869819 retry_after=XXXXs mute_for=86400s (hotfix)
```

**При попытке отправить замученному:**
```
INFO - 🔇 MUTED_SKIP chat_id=1580869819 seconds_left=XXXXX
```

**Статистика (каждую минуту):**
```
INFO - 📊 Mute stats: muted_count=X, retryafter_per_min=Y, top_muted=[...]
```

**Очистка памяти (каждый час):**
```
INFO - 🧹 Cleaned up X old mute state entries
```

## 🔍 Что произойдёт с пользователями

### Сразу после перезапуска:
- **99% пользователей** - никаких изменений, всё работает как раньше
- **Проблемный chat_id 1580869819** - если получит `TelegramRetryAfter > 60 секунд`, будет замучен на 24 часа

### Кто может временно перестать получать сообщения:

1. **chat_id 1580869819** - если получит `retry_after > 60 секунд` → мут на 24 часа
2. **Любой пользователь** - если получит `retry_after > 10 секунд` → мут на 1 час (первый раз)
3. **Любой пользователь** - если получит 3 подряд обычные ошибки отправки → мут на 1 час

**Важно:** Это защита для остальных пользователей. Без мута один проблемный пользователь блокирует доставку всем.

---

## ✅ Подтверждение отсутствия блокировок

### Нет долгих sleep:
- ✅ Для `retry_after > 10 секунд` - **НЕТ sleep**, сразу мут и возврат (0 секунд)
- ✅ Для `retry_after ≤ 10 секунд` - sleep только на короткое время (макс 10 секунд)
- ✅ Для обычных ошибок - sleep только 1 секунда между попытками

### Нет блокировок event loop:
- ✅ `send_trade_alert()` вызывается через `asyncio.create_task()` - не блокирует
- ✅ `_periodic_cleanup()` работает в фоновой задаче - не блокирует
- ✅ Все `asyncio.sleep()` - асинхронные, не блокируют event loop

### При ошибках доставка ухудшается локально, а не глобально:
- ✅ Проблемный пользователь замучивается → остальные получают сообщения нормально
- ✅ Без мута: один пользователь с `retry_after=7000` блокировал бы всех на 2 часа
- ✅ С мутом: проблемный пользователь пропускается, остальные работают
